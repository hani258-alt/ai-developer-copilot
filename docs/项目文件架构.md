# 项目文件架构

## 1.项目结构

- ### src/:存放核心代码

- ### data/:存放文档数据

- ### config/:存放配置文件



## 2.  .env环境配置文件（安全管理敏感信息）

####    在开发中，API 密钥等敏感信息不能硬编码到代码中，需要通过环境变量管理。python-dotenv 库可以从 .env 文件加载环境变量。



## 3.Streamlit 是一个用于快速构建数据应用的 Python 库，可以轻松创建 Web前端界面。



如一个基于RAG的企业知识库问答系统

LangChain trae/
├── src/
│   ├── rag_utils.py      # RAG 核心工具函数
│   └── app.py            # Streamlit 前端界面
├── data/
│   └── sample_doc.txt    # 示例文档
├── config/               # 配置文件夹
├── .env                  # 环境变量（API密钥）
├── main.py               # 命令行入口
└── requirements.txt      # 依赖列表



## 4.  .idea文件夹（一般在.gitignore里）

###       是JetBrains系列IDE自动生成的项目配置目录

### 里面存放了当前项目的工作区设置、代码风格、运行配置、版本控制映射等信息



**常见内容示例**

- misc.xml:项目级别的IDE设置（只对当前这个项目生效的配置）
- modules.xml:模块结构信息
- workspase.xml:本地工作区状态（如打开的文件、断点、窗口布局）
- vcs.xml：版本控制配置（Git等）





## 一、版本控制与协作

| 文件/文件夹      | 含义                                                     |
| :--------------- | :------------------------------------------------------- |
| `.git/`          | Git 版本控制系统的本地仓库目录（自动生成，一般忽略）     |
| `.gitignore`     | 列出不应被 Git 追踪的文件/文件夹（如编译输出、本地配置） |
| `.gitattributes` | 设置 Git 对特定文件的行为（如换行符、合并策略）          |
| `README.md`      | 项目介绍、安装、使用说明（Markdown 格式）                |
| `LICENSE`        | 项目许可证（如 MIT、Apache 2.0）                         |
| `CHANGELOG.md`   | 版本更新日志                                             |

------

## 二、依赖与包管理

| 文件/文件夹                                          | 含义                                            |
| :--------------------------------------------------- | :---------------------------------------------- |
| `package.json`                                       | Node.js 项目依赖、脚本、元信息（npm/yarn/pnpm） |
| `package-lock.json` / `yarn.lock` / `pnpm-lock.yaml` | 锁定依赖版本，确保环境一致                      |
| `.npmrc`                                             | npm 配置（如 registry、代理）                   |
| `requirements.txt`                                   | Python 项目依赖（pip 格式）                     |
| `pyproject.toml`                                     | Python 项目构建系统及依赖（PEP 518/621）        |
| `Pipfile` / `Pipfile.lock`                           | Pipenv 使用的依赖管理文件                       |
| `poetry.lock`                                        | Poetry 依赖锁定文件                             |
| `Cargo.toml` / `Cargo.lock`                          | Rust 项目依赖与构建配置                         |
| `go.mod` / `go.sum`                                  | Go 模块依赖管理                                 |
| `pom.xml`                                            | Maven（Java）项目对象模型：依赖、插件、构建配置 |
| `build.gradle` / `build.gradle.kts`                  | Gradle（Java/Android）构建脚本                  |
| `settings.gradle`                                    | Gradle 多模块项目配置                           |

------

## 三、构建与运行配置

| 文件/文件夹                    | 含义                                          |
| :----------------------------- | :-------------------------------------------- |
| `Makefile`                     | make 构建规则（C/C++/通用自动化）             |
| `CMakeLists.txt`               | CMake 构建配置（跨平台 C++）                  |
| `Dockerfile`                   | 构建 Docker 镜像的指令集                      |
| `docker-compose.yml`           | 定义多容器 Docker 应用的服务、网络、卷        |
| `.dockerignore`                | 构建 Docker 镜像时忽略的文件                  |
| `webpack.config.js`            | Webpack 打包配置（JavaScript 项目）           |
| `vite.config.js`               | Vite 构建工具配置                             |
| `tsconfig.json`                | TypeScript 编译选项（目标、模块、严格模式等） |
| `.babelrc` / `babel.config.js` | Babel（JavaScript 转译）配置                  |
| `next.config.js`               | Next.js 框架配置                              |
| `vue.config.js`                | Vue CLI 配置                                  |

------

## 四、环境与运行时

| 文件/文件夹                    | 含义                                   |
| :----------------------------- | :------------------------------------- |
| `.env`                         | 环境变量（通常包含密钥、端口，不提交） |
| `.env.example` / `.env.sample` | 环境变量模板（提交到仓库）             |
| `.env.local`                   | 本地覆盖配置（不提交）                 |
| `Procfile`                     | Heroku / Cloud Foundry 启动命令定义    |

------

## 五、代码质量与规范

| 文件/文件夹                       | 含义                                          |
| :-------------------------------- | :-------------------------------------------- |
| `.editorconfig`                   | 统一编辑器缩进、换行符等基础规范              |
| `.eslintrc.js` / `.eslintrc.json` | ESLint（JavaScript/TS 代码检查）配置          |
| `.prettierrc`                     | Prettier（代码格式化）配置                    |
| `.stylelintrc`                    | CSS/SCSS 代码检查配置                         |
| `.flake8` / `setup.cfg`           | Python 代码检查（flake8、isort 等）配置       |
| `.pylintrc`                       | Pylint（Python）配置                          |
| `ruff.toml`                       | Ruff（快速 Python linter）配置                |
| `clang-format` / `.clang-format`  | C/C++/ObjC 代码格式化                         |
| `commitlint.config.js`            | Git 提交信息规范检查                          |
| `husky/`                          | Git 钩子管理目录（pre-commit, commit-msg 等） |
| `.lintstagedrc`                   | 仅对暂存文件运行 linter                       |

## 六、IDE 与编辑器

| 文件/文件夹               | 含义                                                         |
| :------------------------ | :----------------------------------------------------------- |

| `.project` / `.classpath` | Eclipse 项目描述文件（Java）                                 |

## 八、特定语言/框架

| 文件/文件夹                       | 含义                               |
| :-------------------------------- | :--------------------------------- |
| `manage.py`                       | Django 项目的命令行工具入口        |
| `wsgi.py` / `asgi.py`             | Python Web 应用部署入口            |
| `__init__.py`                     | 标记 Python 包目录（可为空）       |
| `index.html`                      | 网页入口文件                       |
| `app.js` / `main.js` / `index.js` | Node.js 或前端入口文件             |
| `Main.java`                       | Java 主类文件（命名可不同）        |
| `AndroidManifest.xml`             | Android 应用清单（权限、组件声明） |
| `Podfile`                         | iOS/macOS CocoaPods 依赖管理       |

------

## 九、文档与数据

| 文件/文件夹 | 含义                         |
| :---------- | :--------------------------- |
| `docs/`     | 项目文档目录                 |
| `examples/` | 示例代码目录                 |
| `data/`     | 静态数据文件（CSV、JSON 等） |
| `scripts/`  | 辅助脚本（部署、数据迁移等） |

------

## 十、常见顶级文件夹（非文件）

| 文件夹                      | 含义                                     |
| :-------------------------- | :--------------------------------------- |
| `src/`                      | 源代码主目录                             |
| `lib/`                      | 库文件或第三方依赖（部分项目）           |
| `test/` / `tests/`          | 单元测试/集成测试代码                    |
| `build/` / `dist/` / `out/` | 编译输出/打包产物（通常被 `.gitignore`） |
| `node_modules/`             | npm 依赖安装目录（不提交）               |
| `venv/` / `env/` / `.venv`  | Python 虚拟环境目录（不提交）            |
